## 🛠️ 关于“生成 2 个单元”的技术原理解释

目前在 `src/agents/pipeline.ts` 的 `generatePlan` 核心方法中，Prompt 存在如下**硬编码限制**：

```typescript
You are FCAgent CurriculumPlanner. Generate a personalized learning curriculum array (JSON) with exactly 2 units based on the learner profile.
```

* **为什么设置 2 个单元？**
  在大模型开发初期，为了**控制 API Token 消耗**、**缩短网络等待时间**以及**避免由于 LLM 上下文溢出导致的 JSON 解析格式失败**，系统故意将学习大纲限制在了 2 个单元。
* **优化思路：根据学习时长动态调整大纲规模**

  1. **输入收集**：在诊断命令 (`fc diagnose`) 的提问阶段，新增收集：`totalWeeks` (总学习时长/周)。
  2. **公式计算**：在规划阶段 (`fc plan`)，根据收集到的 `totalWeeks` 与每周可投入时间 `weeklyHours` 计算出一个期望生成的单元总数：
     $$
     \text{期望单元数} = \min\left(10, \max\left(2, \frac{\text{总周数} \times \text{每周小时数}}{5}\right)\right)
     $$
  3. **动态 Prompt**：将 Prompt 中硬编码的 `exactly 2 units` 替换为插值 `exactly ${targetUnitCount} units`，大模型就能根据你的学习时间动态生成更长、更丰富的课程表！
