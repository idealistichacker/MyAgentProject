# 数组与双指针：用两个游标压缩无效操作

## 1. 为什么需要双指针？

如果你要在有序数组里删除重复元素，最直观的方法可能是：遍历数组、遇到重复就删除、继续向后检查。
但数组删除元素会让后面的元素整体移动，成本很高。双指针的核心思想是：不要真的删除，而是把需要保留的元素紧凑地搬到数组前面。

## 2. slow / fast 指针模型

- `fast`：负责向前探索，查看新元素
- `slow`：指向下一个应该写入保留元素的位置

当 `nums[fast] !== nums[slow]` 时，说明发现了一个新值：slow 向前移动一格，然后把 `nums[fast]` 写到 `nums[slow]`。
最后 `slow + 1` 就是去重后有效部分的长度。

## 3. 示例

输入：`[1, 1, 2, 2, 3]`。
过程：fast 从 1 开始扫描；遇到重复值就跳过；遇到新值就写入 slow 的下一个位置。
最终前三个元素变为 `[1, 2, 3]`，返回长度 3。

## 4. 代码模板

```ts
export function removeDuplicates(nums: number[]): number {
  if (nums.length === 0) return 0;
  let slow = 0;
  for (let fast = 1; fast < nums.length; fast++) {
    if (nums[fast] !== nums[slow]) {
      slow++;
      nums[slow] = nums[fast];
    }
  }
  return slow + 1;
}
```
